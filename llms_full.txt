## Project Overview

C++23 DB driver library built on qbuem-stack.
Async ORM + 4 drivers (PostgreSQL · MySQL · SQLite3 · Redis) + migration tool.

## src/db/orm.hpp — Header-Only ORM (Driver-Independent)

C++23 Concepts-based. `TableMeta<T>` provides SQL generation + binding + reading methods.
Dialect setting enables compatibility with all drivers.

### Dialect
```
enum class Dialect {
    PostgreSQL  (default / $N placeholders / RETURNING)
    MySQL       (? placeholders / no RETURNING / ON DUPLICATE KEY)
    SQLite      (? placeholders / no RETURNING / ON CONFLICT EXCLUDED)
}
TableMeta<T>& dialect(Dialect d)  — set in builder chain
```

### Supported Type Concepts
- `OrmInt`:      `int16_t` / `int32_t` / `int64_t` / `uint32_t`
- `OrmFloat`:    `double` / `float`
- `OrmText`:     `std::string`
- `OrmBool`:     `bool`
- `OrmOptional`: `std::optional<OrmScalar>`

### TableMeta<T> SQL Generation Methods (complete)
```
sql_select_all()                                    SELECT col1,… FROM table
sql_select_where(col, p=1)                          SELECT … WHERE col=$p / ?
sql_select_ordered(col, order_col, p, sort)         WHERE col=$p ORDER BY order_col [ASC|DESC]
sql_select_all_ordered(order_col, sort)             ORDER BY order_col [ASC|DESC]
sql_select_raw_where(raw_where)                     WHERE <raw_where>
sql_select_where_in(col, n, start_p=1)             WHERE col IN ($1,…,$n) / (?,…,?)
sql_select_where_null(col)                          WHERE col IS NULL
sql_select_where_not_null(col)                      WHERE col IS NOT NULL
sql_select_where_between(col, p1=1, p2=2)          WHERE col BETWEEN $1 AND $2 / ? AND ?
sql_select_where_like(col, p=1)                     WHERE col LIKE $1 / ?
sql_select_paged(limit_p=1, offset_p=2)            LIMIT $l OFFSET $o
sql_select_where_paged(col, wp=1, lp=2, op=3)      WHERE col=$1 LIMIT $2 OFFSET $3
sql_select_all_ordered_paged(order_col, sort, l, o) ORDER BY … LIMIT $l OFFSET $o
sql_count()                                         SELECT COUNT(*) FROM table
sql_count_where(col, p=1)                           SELECT COUNT(*) … WHERE col=$p
sql_count_raw_where(raw_where)                      SELECT COUNT(*) … WHERE raw
sql_count_where_in(col, n, start_p=1)              SELECT COUNT(*) … WHERE col IN (…)
sql_select_where2(c1, c2, p1=1, p2=2)              WHERE c1=$1 AND c2=$2
sql_select_where3(c1, c2, c3)                       WHERE c1=$1 AND c2=$2 AND c3=$3
sql_select_where_multi(span<string_view>)           WHERE c1=$1 AND c2=$2 AND …
sql_insert(returning=true)                          INSERT INTO … VALUES (…) [RETURNING *]
sql_insert_batch(n, returning=true)                 INSERT … VALUES (…),(…),… [RETURNING *]
sql_upsert_pk(returning=true)                       PG/SQLite: ON CONFLICT EXCLUDED / MySQL: ON DUPLICATE KEY VALUES

sql_delete_pk()                                     DELETE … WHERE pk=$1
sql_delete_where(col)                               DELETE … WHERE col=$1
sql_delete_where2(c1, c2)                           DELETE … WHERE c1=$1 AND c2=$2
```

### TableMeta<T> Binding Methods
```
bind_insert(obj)                → vector<Value>  excludes PK
bind_update(obj)                → vector<Value>  non-PK first, PK last
bind_batch(span<T>)             → vector<Value>  flat params for batch INSERT
bind_pk(obj)                    → vector<Value>  PK only
bind_val(v)                     → vector<Value>  single value
bind_val2(v1, v2)               → vector<Value>  2 values
bind_val3(v1, v2, v3)           → vector<Value>  3 values
bind_paged(where_val, lim, off) → vector<Value>  WHERE+LIMIT+OFFSET
bind_in(range<OrmScalar>)       → vector<Value>  IN clause (vector/span/init-list)
```

```

```

## src/db/postgresql_driver.hpp/cpp — Async PostgreSQL

Factory: `make_postgresql_driver()` → `unique_ptr<IDBDriver>`
DSN: `postgresql://user:pass@host:port/db` / `postgresql:///db` (Unix socket)

### Architecture
```
PgDriver → PgConnectionPool → Slot{PGconn*, mutex, stmt_counter}
PgConnection / PgTransaction / PgStatement / PgRow / PgResultSet
```

### Async Awaiters
- `PgFlushAwaiter`: yields on `PQflush()==1` until socket is writable
- `PgReadAwaiter`:  yields on `PQisBusy==true` until socket is readable; resumes via `Reactor::post(resume)`
- `PgAcquireAwaiter`: yields when pool is exhausted; slot handed directly on `release()`

### Pool Behavior
1. Idle slot → O(1) return
2. slots < max_size → create new PGconn
3. Pool exhausted → `PgAcquireAwaiter` yield

### Binary Protocol (resultFormat=1)
OID 16(bool)→Bool, 20/21/23(int)→Int64, 700/701(float)→Float64,
17(bytea)→Blob, others→Text. Big-endian deserialization via `std::byteswap` (C++23).

## src/db/mysql_driver.hpp/cpp — MySQL/MariaDB

Factory: `make_mysql_driver()` → `unique_ptr<IDBDriver>`
DSN: `mysql://user:pass@host:port/db[?ssl=true]` / `mariadb://…`

### Features
- `$N` → `?` placeholder conversion (`convert_placeholders`)
- libmysqlclient + `MYSQL_STMT` prepared statements
- Multi-connection pool (`MysqlSlot{MYSQL*, mutex}` per connection)
- UTF8MB4 charset, TLS/SSL, `MYSQL_OPT_RECONNECT`
- `MysqlDriver` / `MysqlConnectionPool` / `MysqlConnection` /
  `MysqlTransaction` / `MysqlStatement` / `MysqlRow` / `MysqlResultSet`

### Cloud Compatibility
RDS MySQL · Aurora MySQL · Cloud SQL · Azure DB for MySQL ·
PlanetScale · TiDB Cloud · MariaDB SkySQL

## src/db/redis_client.hpp/cpp — Async Redis Client

DSN: `redis://[:pass@]host[:port][/db]`
Pure POSIX TCP + RESP2. No external dependencies. Fully integrated with qbuem Reactor.

### Architecture
```
RedisDsn parser → connect_tcp → RedisConnectAwaiter
RedisClientImpl{fd, RespParser}
RedisWriteAwaiter: non-blocking write + writable event
RedisReadAwaiter:  readable event until RespParser has complete response
RespParser: RESP2 streaming parser (has_complete + parse)
```

### Supported Commands
```
Strings:     GET/SET(TTL)/GETSET/SETNX/DEL/EXISTS/EXPIRE/TTL/PTTL/INCR/INCRBY/DECR/DECRBY/MGET/MSET
Hash:        HGET/HSET/HSETNX/HDEL/HEXISTS/HGETALL/HKEYS/HVALS/HLEN
List:        LPUSH/RPUSH/LPOP/RPOP/LRANGE/LLEN/LINDEX
Set:         SADD/SREM/SMEMBERS/SCARD/SISMEMBER
Sorted Set:  ZADD/ZREM/ZRANGE/ZRANGEBYSCORE/ZSCORE/ZCARD/ZRANK
Other:       TYPE/KEYS/SCAN/RENAME/PERSIST/PING/FLUSHDB/INFO + command(args)
```

### Cloud Compatibility
ElastiCache · Upstash · Redis Cloud · Memorystore · Azure Cache

## src/db/migration/migrator.hpp — Migration Framework

Header-only. Operates on the `IConnection` interface. Driver-independent.

### Structs
```
Migration{version, description, up, down}
MigrationStatus{version, description, applied, applied_at}
MigrationResult{applied, skipped, latest}
PlaceholderStyle{Dollar($N), Question(?)}
```

### MigrationRunner API
```
MigrationRunner(migrations, conn, style=Dollar)
migrate()            → MigrationResult   apply all pending
migrate_to(ver)      → MigrationResult   apply up to version
rollback()           → void              roll back latest one
rollback_to(ver)     → void              roll back all above version
status()             → vector<MigrationStatus>
is_applied(ver)      → bool
current_version()    → uint64_t
```

History table: `__schema_migrations{version BIGINT PK, description TEXT, applied_at TIMESTAMP}`
Multi-statement SQL via semicolon splitting (`split_sql`). Atomic application within a transaction.
