## 프로젝트 개요

C++23 기반 DB 드라이버 라이브러리. qbuem-stack 위에서 동작.
비동기 ORM + 4개 드라이버(PostgreSQL·MySQL·SQLite3·Redis) + 마이그레이션 툴.

## src/db/orm.hpp — 헤더 전용 ORM

C++23 Concepts 기반. TableMeta<T>에 SQL 생성 + 바인딩 + 읽기 메서드.

### 지원 타입 Concepts
- OrmInt:      int16_t / int32_t / int64_t / uint32_t
- OrmFloat:    double / float
- OrmText:     std::string
- OrmBool:     bool
- OrmOptional: std::optional<OrmScalar>

### TableMeta<T> SQL 생성 메서드 (전체)
sql_select_all()                                    SELECT col1,… FROM table
sql_select_where(col, p=1)                          SELECT … WHERE col=$p
sql_select_ordered(col, order_col, p, sort)         WHERE col=$p ORDER BY order_col [ASC|DESC]
sql_select_all_ordered(order_col, sort)             ORDER BY order_col [ASC|DESC]
sql_select_raw_where(raw_where)                     WHERE <raw_where>
sql_select_paged(limit_p=1, offset_p=2)            LIMIT $l OFFSET $o
sql_select_where_paged(col, wp=1, lp=2, op=3)      WHERE col=$1 LIMIT $2 OFFSET $3
sql_select_all_ordered_paged(order_col, sort, l, o) ORDER BY … LIMIT $l OFFSET $o
sql_count()                                         SELECT COUNT(*) FROM table
sql_count_where(col, p=1)                           SELECT COUNT(*) … WHERE col=$p
sql_count_raw_where(raw_where)                      SELECT COUNT(*) … WHERE raw
sql_select_where2(c1, c2, p1=1, p2=2)              WHERE c1=$1 AND c2=$2
sql_select_where3(c1, c2, c3)                       WHERE c1=$1 AND c2=$2 AND c3=$3
sql_select_where_multi(span<string_view>)           WHERE c1=$1 AND c2=$2 AND …
sql_insert(returning=true)                          INSERT INTO … VALUES (…) [RETURNING *]
sql_insert_batch(n, returning=true)                 INSERT … VALUES (…),(…),… [RETURNING *]
sql_upsert_pk(returning=true)                       INSERT … ON CONFLICT(pk) DO UPDATE SET …
sql_update_pk(returning=true)                       UPDATE … SET … WHERE pk=$N [RETURNING *]
sql_update_col_pk(col, returning=true)              UPDATE … SET col=$1 WHERE pk=$2
sql_delete_pk()                                     DELETE … WHERE pk=$1
sql_delete_where(col)                               DELETE … WHERE col=$1
sql_delete_where2(c1, c2)                           DELETE … WHERE c1=$1 AND c2=$2

### TableMeta<T> 바인딩 메서드
bind_insert(obj)                → vector<Value>  PK 제외
bind_update(obj)                → vector<Value>  non-PK + PK 마지막
bind_batch(span<T>)             → vector<Value>  배치 INSERT용 flat
bind_pk(obj)                    → vector<Value>  PK만
bind_val(v)                     → vector<Value>  단일 값
bind_val2(v1, v2)               → vector<Value>  2개
bind_val3(v1, v2, v3)           → vector<Value>  3개
bind_paged(where_val, lim, off) → vector<Value>  WHERE+LIMIT+OFFSET

### 전역 함수
orm::register_table<T>(name) → TableMeta<T>&   등록 + 빌더 반환
orm::meta<T>()               → const TableMeta<T>&  등록된 메타 접근
orm::is_registered<T>()      → bool

## src/db/postgresql_driver.hpp/cpp — 비동기 PostgreSQL

팩토리: make_postgresql_driver() → unique_ptr<IDBDriver>
DSN: postgresql://user:pass@host:port/db  /  postgresql:///db (Unix socket)

### 아키텍처
PgDriver → PgConnectionPool → Slot{PGconn*, mutex, stmt_counter}
PgConnection / PgTransaction / PgStatement / PgRow / PgResultSet

### 비동기 Awaiter
PgFlushAwaiter: PQflush()==1 시 writable 이벤트 yield
PgReadAwaiter:  PQisBusy==true 시 readable 이벤트 yield, Reactor::post(resume)
PgAcquireAwaiter: 풀 소진 yield → release() 시 슬롯 직접 전달

### 풀 동작
1. idle 슬롯 → O(1) 반환
2. slots < max_size → 새 PGconn 생성
3. 풀 소진 → PgAcquireAwaiter yield

### 바이너리 프로토콜 (resultFormat=1)
OID 16(bool)→Bool, 20/21/23(int)→Int64, 700/701(float)→Float64,
17(bytea)→Blob, others→Text. std::byteswap(C++23) big-endian 역직렬화.

## src/db/redis_client.hpp/cpp — Redis 비동기 클라이언트

DSN: redis://[:pass@]host[:port][/db]
순수 POSIX TCP + RESP2. 외부 의존성 없음. qbuem Reactor 완전 통합.

### 구조
RedisDsn 파서 → connect_tcp → RedisConnectAwaiter
RedisClientImpl{fd, RespParser}
RedisWriteAwaiter: non-blocking write + writable 이벤트
RedisReadAwaiter:  RespParser 완성까지 readable 이벤트
RespParser: RESP2 스트리밍 파서 (has_complete + parse)

### 지원 명령
문자열: GET/SET(TTL)/SETNX/DEL/EXISTS/EXPIRE/TTL/INCR/INCRBY/DECR/DECRBY/MGET
해시:   HGET/HSET/HSETNX/HDEL/HEXISTS/HGETALL/HLEN
리스트: LPUSH/RPUSH/LPOP/RPOP/LRANGE/LLEN
셋:    SADD/SREM/SMEMBERS/SCARD/SISMEMBER
정렬셋: ZADD/ZREM/ZRANGE/ZSCORE/ZCARD/ZRANK
기타:  TYPE/KEYS/INFO/FLUSHDB/PING + command(args)

### 클라우드 호환
ElastiCache · Upstash · Redis Cloud · Memorystore · Azure Cache

## src/db/migration/migrator.hpp — 마이그레이션 프레임워크

헤더 전용. IConnection 인터페이스 위에서 동작. 드라이버 독립.

### 구조체
Migration{version, description, up, down}
MigrationStatus{version, description, applied, applied_at}
MigrationResult{applied, skipped, latest}
PlaceholderStyle{Dollar($N), Question(?)}

### MigrationRunner API
MigrationRunner(migrations, conn, style=Dollar)
migrate()            → MigrationResult   미적용 전부
migrate_to(ver)      → MigrationResult   특정 버전까지
rollback()           → void              최신 1개 롤백
rollback_to(ver)     → void              버전 초과분 전부 롤백
status()             → vector<MigrationStatus>  현황
is_applied(ver)      → bool
current_version()    → uint64_t

이력: __schema_migrations{version BIGINT PK, description TEXT, applied_at TIMESTAMP}
세미콜론 구분 다중 SQL (split_sql), 트랜잭션 내 원자 적용.
